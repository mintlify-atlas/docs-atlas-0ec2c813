Development modules live across the [`base`](/modules/home/overview#base) and [`userspace`](/modules/home/overview#userspace) profiles. Core VCS and language tools ship in `base` so they are available on all hosts. Graphical editors and desktop-only tools are part of `userspace`.

## Helix

Helix is the default editor. The `defaultEditor` option sets the `EDITOR` and `VISUAL` environment variables so programs that invoke an editor (Git, Jujutsu, `sudoedit`, etc.) open Helix automatically.

```nix modules/home/helix.nix
{ ... }:
{
  programs.helix = {
    enable = true;
    defaultEditor = true;
  };
}
```

## VS Code

VS Code is configured with a full set of extensions sourced from `pkgs.vscode-marketplace` (provided by [nix-vscode-extensions](https://github.com/nix-community/nix-vscode-extensions)), custom keybindings, and detailed editor settings. Update checks are disabled because updates are managed through Nix.

## Zed

Zed is configured with the Catppuccin Mocha theme, Nix language support, and custom keybindings that mirror the VS Code layout. A `zed` shell alias resolves to the full package path.

```nix modules/home/zed.nix
{ lib, pkgs, ... }:
{
  home.shellAliases.zed = lib.getExe pkgs.zed-editor;

  programs.zed-editor = {
    enable = true;

    extensions = [
      "catppuccin"
      "nix"
    ];

    userSettings = {
      theme = "Catppuccin Mocha";
      auto_signature_help = true;
      disable_ai = true;
      diagnostics.inline.enabled = true;
      wrap_guides = [ 100 ];

      languages = {
        Nix.formatter = "language_server";
        QML.formatter = "language_server";
        Rust.formatter = "language_server";
      };
    };
  };
}
```

Key settings: AI is disabled, inline diagnostics are enabled, the wrap guide sits at 100 columns, and language servers handle formatting for Nix, QML, and Rust.

<Note>
  Zed is excluded from Stylix theming (`stylix.targets.zed.enable = false`). The Catppuccin Mocha theme is set via `userSettings.theme`.
</Note>

## Jujutsu

Jujutsu (`jj`) is the primary version control system. The module sets the user identity used in commits.

```nix modules/home/jujutsu.nix
{ ... }:
{
  programs.jujutsu = {
    enable = true;

    settings.user = {
      name = "jel";
      email = "25802745+jelni@users.noreply.github.com";
    };
  };
}
```

The email is a GitHub no-reply address, which keeps the real address private while still associating commits with the GitHub account.

## Jujutsu signing

Commit signing is configured separately from the core Jujutsu module so it can be composed differently between bare-metal and WSL hosts. On bare-metal, `op-ssh-sign` from the native 1Password installation is used. On WSL, `op-ssh-sign.exe` is used instead (see [WSL-specific tools](#wsl-specific-tools) below).

```nix modules/home/jujutsu-signing.nix
{ ... }:
{
  programs.jujutsu.settings = {
    signing = {
      behavior = "force";
      backend = "ssh";
      key = "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIIeyqBh/je0A8dMXB0m9jxZ0nBWHc5Hj+dFmFBiT5Mdw";
    };

    git.sign-on-push = true;
  };
}
```

All commits are force-signed with the ed25519 SSH key managed by 1Password. `git.sign-on-push = true` ensures signatures are added when pushing to Git-backed remotes.

## WSL-specific tools

Two modules handle the boundary between the Linux WSL environment and the Windows host.

<Tabs>
  <Tab title="wsl-git">
    `wsl-git` routes Git's SSH command to the Windows `ssh.exe` binary. This lets Git use the Windows OpenSSH agent (or 1Password agent on Windows) for authentication without any additional configuration inside WSL.

    ```nix modules/home/wsl-git.nix
    { ... }:
    {
      programs.git.settings.core.sshCommand = "ssh.exe";
    }
    ```
  </Tab>
  <Tab title="wsl-1password">
    `wsl-1password` overrides the Jujutsu SSH signing program to use the Windows `op-ssh-sign.exe` binary. This is applied on top of the [`jujutsu-signing`](#jujutsu-signing) module, which sets the key and signing behaviour, so only the binary path needs to change.

    ```nix modules/home/wsl-1password.nix
    { ... }:
    {
      programs.jujutsu.settings.signing.backends.ssh.program = "op-ssh-sign.exe";
    }
    ```

    This module is part of the optional [`wsl-1password`](/modules/home/overview#wsl-1password) profile. Hosts that use WSL without 1Password on Windows skip this module.
  </Tab>
</Tabs>
